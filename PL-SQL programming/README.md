# PL-SQL Programming

This module contains hands-on exercises that demonstrate the fundamentals and advanced features of **PL/SQL (Procedural Language/Structured Query Language)**. The exercises focus on developing robust database programs using control structures, stored procedures, functions, exception handling, triggers, cursors, and packages.

---

## 📖 Overview

PL/SQL extends SQL with procedural programming capabilities, enabling developers to write efficient, secure, and reusable database applications. This module provides practical implementations of essential PL/SQL concepts through real-world exercises.

---

## 🎯 Objectives

- Understand the fundamentals of PL/SQL programming.
- Write procedural database programs using SQL and PL/SQL.
- Create reusable stored procedures and functions.
- Handle runtime exceptions effectively.
- Automate database operations using triggers.
- Process multiple records efficiently using cursors.
- Organize related procedures and functions using packages.

# 📚 Exercises

## Mandatory Hands-on

### Exercise 1 – Control Structures

**Concepts Covered**

- Conditional Statements (IF, CASE)
- Looping Statements (FOR, WHILE, LOOP)
- Variable Declaration
- PL/SQL Blocks

**Objective**

Learn how to control the execution flow of PL/SQL programs using conditional and iterative statements.

---

### Exercise 3 – Stored Procedures

**Concepts Covered**

- Creating Stored Procedures
- Input and Output Parameters
- Procedure Invocation
- Reusability

**Objective**

Develop reusable database procedures to simplify business logic execution.

---

## Additional Important Hands-on

### Exercise 2 – Exception Handling

**Concepts Covered**

- Predefined Exceptions
- User-Defined Exceptions
- Exception Propagation
- Exception Handling Block

**Objective**

Handle runtime errors gracefully and improve application reliability.

---

### Exercise 4 – Functions

**Concepts Covered**

- User-Defined Functions
- Return Values
- Parameter Passing
- SQL Integration

**Objective**

Create reusable functions that return computed values and can be called from SQL queries.

---

### Exercise 5 – Triggers

**Concepts Covered**

- BEFORE Triggers
- AFTER Triggers
- INSERT, UPDATE, DELETE Triggers
- Database Automation

**Objective**

Automate database operations by executing PL/SQL code in response to table events.

---

### Exercise 6 – Cursors

**Concepts Covered**

- Implicit Cursors
- Explicit Cursors
- Cursor FOR Loops
- Record Processing

**Objective**

Efficiently retrieve and process multiple rows returned by SQL queries.

---

### Exercise 7 – Packages

**Concepts Covered**

- Package Specification
- Package Body
- Modular Programming
- Encapsulation

**Objective**

Organize related procedures, functions, variables, and cursors into reusable packages.
